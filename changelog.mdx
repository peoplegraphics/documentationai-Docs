Release notes for the PanelWave **format** and this **documentation site**. For the format's design and compatibility rules, see [Versioning](/schema/versioning).

<Update label="2026-07-24" description="Format 1.4.0 · The Infinite Canvas — schema, player & CMS" tags={["feature"]}>
  ## The infinite canvas: one continuous plane, a camera on the trail

  Format **1.4.0** brings the *infinite canvas* to the whole
  PanelWave stack: a chapter's panels can live on one continuous world-space
  plane, and the reader's screen becomes a camera that glides between them
  along the story graph. A single-column layout is a webtoon-style vertical
  strip; free 2D placement with camera trails is the full experience. All
  additive — works without a canvas are untouched, and print keeps using
  pages. See [The Infinite Canvas](/concepts/infinite-canvas).

  **Format (schema 1.4.0)** — all additive; see [Infinite Canvas](/schema/canvas):
  - **`Chapter.canvas`** — world-unit panel `placements` (1 wu = 1 CSS px at
    zoom 1; rotation, stacking, panel-relative `enterFraming`), `revealMode`
    spoiler protection (`always` / `on-approach` / `on-visit`), plane
    `background`, camera policy (`fitMode`, `overview`, `freeRoam`,
    `bounds`), and presentational `decorations` with parallax depth.
  - **`Edge.cameraMove`** — camera travel per graph edge: `direct` / `arc` /
    `waypoints` paths, `hold` / `pull-back` / `dive` zoom profiles, duration,
    easing, and a `reducedMotionFallback` transition. Inherits like
    `transition` via the new `FormatPreset.defaultCameraMove`.
  - **`FormatPreset.canvasView`** — per-format opt-in; players without it (or
    without canvas support) fall back to panel view.
  - `@panelwave/cli` validates the new semantic rules (placement keys must be
    chapter panels, performance budget, waypoint consistency) and its
    `upgrade` target is now synchronized at `1.4.0`.

  **Player** — a third view mode alongside panel and page view; see
  [Canvas View](/player/canvas-view):
  - Camera glides with authored paths and zoom profiles (motion-safety speed
    cap, interruptible by the reader), overview zoom (`O`), guided or
    free-roam input on touch, mouse, and keyboard.
  - Virtualized rendering (≤ 12 mounted panels regardless of chapter size),
    **variant-by-zoom** image selection (overview mounts thumbnails,
    close-ups load full resolution, upgrade-only — no flicker), and
    settle-time neighbor preloading honoring `settings.preload`.
  - Accessibility built in: no gliding under reduced motion (cross-fade
    reframes instead), full keyboard traversal, `aria-hidden`/`inert` on
    non-current panels, graph-driven reading order.
  - New embedding API: `viewModeOverride` input and throttled `cameraChange`
    output.

  **CMS** — a new authoring surface; see
  [Canvas Layout Board](/cms/editor/canvas-layout):
  - Spatial board with live panel artwork, graph-edge arrows, drag/resize
    with snap, align/distribute, and auto-layout presets (**vertical strip**
    for webtoon-shaped works, layered **graph layouts** for branching 2D
    canvases).
  - Camera-move inspector with **path ghosting** and draggable waypoints,
    **gap calipers** showing distance-as-time in beats, an **enter-framing**
    tool, per-panel reveal modes, and decoration authoring (opacity,
    parallax, rotation).
  - Work Properties → Output gains per-format **Infinite Canvas View**
    toggles; publish preflight validates canvas integrity; the
    [Preview](/cms/preview) tab's new **View** selector compares panel and
    canvas modes on the live manifest.
</Update>

<Update label="2026-07-17" description="CMS · AI page import, upscaling credits, publish pipeline live" tags={["feature", "bugfix"]}>
  ## Import finished pages, upscale panels, and publish for real

  - **Page Import (AI panel splitting)** — a new importer for artists who already
    have finished works as flat page images. Upload PNG/JPEG/WebP pages, AI
    detects the panels (regular grids, full-bleed artwork, and nested inset
    panels) with a selectable **EU/US or manga reading direction**, and a review
    step lets you move/resize/add/delete panels and fix the reading order before
    anything is created. On import every panel is cropped borderless from the
    full-resolution page, saved as an asset, and placed at its original position
    on a PanelWave page — with a linear reading graph so the work is playable
    immediately. See [Page Import](/cms/page-import).
  - **AI upscaling with credits** — optionally enlarge small panel crops for
    sharp desktop/4K display (Standard 1440 / High 2560 / Ultra 3840 px
    targets). Upscaling is billed in **credits**, PanelWave's new prepaid wallet
    for metered AI features: plans include a monthly allowance, the wizard shows
    the exact cost against your balance before you commit, and panels that fail
    to upscale import at original resolution with their reserved credits
    returned automatically.
  - **The publish pipeline is live** — the guided Publish flow (preflight →
    environment → options → confirm) now runs end-to-end, and a **Preview**
    publish mints an unguessable, revocable **preview manifest link** that any
    PanelWave Player can load without a CMS login — ideal for sharing a
    work-in-progress with testers. See [Publishing](/cms/publishing).
  - **Size &amp; Content stats** — the publish result and the Work Properties
    Metadata tab now break a work's weight down into manifest size, asset count
    and payload, and the total.
  - **Player demo: external manifests** — the demo app accepts
    `?manifest=<url>`, so a CMS preview link opens directly in the standalone
    player. See [Contributing](/player/development).
</Update>

<Update label="2026-07-16" description="CMS · asset folders, asset picker & extras" tags={["feature", "bugfix"]}>
  ## Organize the library, pick assets anywhere, and upload straight into Extras

  - **Asset folders** — the Assets tab gains a folder pane (nested folders,
    drag-to-file, per-folder counts, an **Unfiled** view). Folders are purely
    organizational and never affect exports.
  - **One asset picker everywhere** — a new **Browse library…** dialog with the
    folder tree on the left, the searchable/filterable library in the center,
    and a large preview with details on the right. It opens pre-filtered to the
    type a field expects, and an in-dialog **Upload** drops new files into the
    selected folder and picks them immediately.
  - **Extras take library assets** — an extra's content and thumbnail can now be
    picked from the library or uploaded on the spot, in addition to external
    URLs. Linked assets appear as a chip (change / clear), auto-fill dimensions,
    duration, and file size, and stay connected — **Replace File** on the asset
    updates the extra automatically.
  - **Characters too** — portrait and reference images gain a **From library**
    button next to upload and URL paste. Page background images/textures and
    video layers use the same picker.
  - **Fix: images no longer expire** — character portraits and extras previously
    stored short-lived signed links that broke ~24 hours after saving. Stored
    links are now durable and re-signed on every read, and published manifests
    always carry fresh asset URLs.

  Details: [Assets](/cms/assets), [Extras](/cms/extras), [Characters](/cms/characters).
</Update>

<Update label="2026-07-16" description="CMS · publish unblockers" tags={["feature", "bugfix"]}>
  ## Optimize oversized images from preflight, and a faster path between work areas

  - **Optimize / Optimize all** — oversized-image errors (over the 5 MB publish
    limit) now carry an **Optimize** button in preflight and in the publish
    wizard. The CMS re-encodes the image as WebP in the background (stepping
    down quality, then resolution, until it fits), keeps your full-quality
    original in the library, and ships the optimized variant. The issue list
    polls until the queued jobs finish and the errors clear on their own;
    **Undo** restores the previous state.
  - **Fair size check** — the oversized check now judges the *effective shipped
    size*, so an already-optimized image isn't flagged again.
  - **Work-areas menu** — a grid-icon dropdown in the editor header jumps
    straight to Characters, Extras, the Graph Editor, Paywall & Entitlement,
    Localization, Audio, and Analytics.
  - **Fixes** — publishing no longer fails with a 404 on the validation call;
    the works list shows creators again; failed optimize jobs can be retried
    immediately instead of silently doing nothing.

  Details: [Validation](/cms/validation#optimizing-oversized-images), [Publishing](/cms/publishing), [Editor Layout](/cms/editor/layout).
</Update>

<Update label="2026-07-13" description="CMS editor · video playback" tags={["feature", "bugfix"]}>
  ## Video panels play in the editor, and better timeline transport

  - **In-editor video playback** — a video panel now plays right on the editing
    canvas. Hover it for a **play / pause / stop** toolbar, and in Panel View the
    footage fills the canvas so you can check framing. Clicks outside the toolbar
    still select and drag the panel as before.
  - **Timeline drives the video** — when you run the timeline, each video panel
    plays automatically while the playhead is inside its time range, seeking to
    match the playhead and pausing when it leaves. Scrubbing while paused shows
    the matching frame, so the timeline preview mirrors reader autoplay.
  - **Jump-to-start / jump-to-end buttons** — the single-arrow buttons step one
    frame (so at the end they barely move); new **⏮ / ⏭** buttons snap the
    playhead to the first or last panel and switch the canvas to it, matching the
    `Home` / `End` keys.
  - **No more autoplay "Loading…" flash** — panel images are preloaded, so
    switching panels during autoplay no longer flashes a grey placeholder while
    each image decodes.
  - **Fixes** — video posters now resolve reliably on the canvas (asset-id
    matching and asset-list variants), and the Preview tab loads its manifest on
    a direct visit instead of showing "No Manifest Available".

  Details: [Video Panels](/cms/editor/video-panels), [Animations & Timeline](/cms/editor/animations).
</Update>

<Update label="2026-07-13" description="Format 1.3.x · CMS editor" tags={["feature", "bugfix"]}>
  ## Narrator caption boxes & format-independent speech bubbles

  - **New `narrator` balloon type** — a true sharp-cornered rectangle for
    narration caption boxes (unlike `rectangle`, which keeps a subtle corner
    rounding). Additive enum extension across schema, `@panelwave/types`, the
    player's ComicBalloon renderer (new `sharpCorners` flag), and the CMS.
  - **Panel-relative bubble geometry** — the CMS now stores a bubble's `shape`
    box exactly as the format defines it: normalized against the **panel
    artwork**, not the editing canvas. Bubbles keep their position on every
    device format, and Panel-Top / narrator caption boxes stay glued to the
    panel border everywhere — panel view, page view, and player.
  - **Per-format lettering size** — balloon text keeps the same comfortable
    reading size on each target screen: unchanged in A4-sized formats,
    moderately larger on big screens, smaller on phones — never proportional
    to the artwork. Page view renders balloons text-fitted, with a readability
    boost capped at 75% of the panel.
  - **Player: per-screen lettering** — the open-source player applies the same
    rule at playback time. Balloons render at their measured, text-fitting
    size scaled by the player viewport (DIN A4 height = 1.0); the stored
    `shape` box is a position anchor with border-flush edges glued, no longer
    a display size. Lettering stays comfortably readable on every device
    instead of blowing up on big screens.
  - **Editor quality of life** — the *Add Speech Bubble* context menu offers
    all ten balloon types including snap-to-border Narrator/Panel-Top
    placements; the panel border is magnetic when dragging caption boxes;
    speech bubbles resize via their corner handles; the canvas backing store
    is HiDPI-aware (crisp lettering on scaled displays); and the editor
    restores your last page/panel view after a browser reload.

  Details: [Speech Bubbles (format)](/schema/speech-bubbles), [Speech Bubbles (editor)](/cms/editor/speech-bubbles), [Balloon Renderer](/player/balloon-renderer), [Canvas](/cms/editor/canvas).
</Update>

<Update label="2026-07-12" description="Format 1.3" tags={["feature"]}>
  ## Format 1.3 — reusable style presets & implicit speech toggle

  Additive, backward-compatible release: every valid 1.0–1.2 manifest remains valid unchanged.

  - **Reusable style presets** — like CSS classes for the manifest. Named text
    styles in `settings.typography.textStyles` (referenced by
    `TextLayer.styleRef`) and named balloon presets in
    `settings.typography.balloonPresets` (referenced by
    `SpeechBubble.styleRef`). Shared styling is defined once and edited in one
    place; the balloon cascade becomes work → character → **preset** → bubble.
    Sample 11 replaced 104 identical caption style blocks and 79 repeated
    balloon overrides with one text style and six balloon presets.
  - **The speech toggle is implicit** — every speech bubble is inherently
    subject to the reader's speech toggle (initial state:
    `settings.ui.speechDefault`). The per-bubble
    `visibleIf: {"var": "prefs.speech"}` boilerplate is obsolete; `visibleIf`
    is reserved for story logic, and a forgotten boilerplate line can no
    longer make a bubble immune to the toggle.
  - **Player** — text layers and balloons resolve `styleRef` presets; the
    toolbar speech toggle is functional and seeded from `speechDefault`.
  - **CMS** — presets are exported with the manifest, `styleRef` round-trips
    through the editor, validation warns about dangling preset references,
    and legacy `prefs.speech` boilerplate is stripped on export.

  Details: [Settings](/schema/settings), [Speech Bubbles](/schema/speech-bubbles), [Layers](/schema/layers), [Versioning](/schema/versioning).
</Update>

<Update label="2026-07-12" description="Format 1.2" tags={["feature"]}>
  ## Format 1.2 — leaner manifests

  Relaxing, backward-compatible release:

  - **`mime` optional** on image/audio/video/subtitle/vector variants —
    consumers derive the MIME type from the `src` file extension.
  - **Edge transition inheritance** — edges without a `transition` inherit the
    active format's `settings.outputPresets[format].defaultTransition`.
  - **Export discipline** — exporters omit values equal to schema defaults.

  Details: [Versioning](/schema/versioning).
</Update>

<Update label="2026-07-06" description="CMS" tags={["feature"]}>
  ## CMS — AI-assisted script import (Fountain)

  Turn a screenplay in the open [Fountain](https://fountain.io/) format into a
  PanelWave work — reviewed by you at every step:

  - **Upload & parse** — scenes, dialogue, and cast are extracted with exact
    source-line provenance; a parse summary shows what was found.
  - **AI structure suggestion** — chapters, panels with art briefs, and typed
    speech bubbles (speech, thought, shout, whisper, caption, narration, SFX).
    Dialogue is guaranteed verbatim: rewritten or dropped lines are detected
    and restored automatically.
  - **Structure review** — a three-pane tree editor with drag & drop, panel
    split/merge, cast merging, a source-line drawer, undo/redo, and autosave.
  - **AI page layouts** — pages laid out per output format (mobile, desktop,
    print, square, video …), refined in a drag & drop preview.
  - **Work creation** — a schema-valid manifest is built and imported
    atomically; speech bubbles come prepositioned, artwork is added behind
    them in the editor afterwards.
  - **Plan-tier metering** — script imports are allowance-gated per
    subscription tier, with per-session retry caps.
  - Resilient by design: if the AI service is unavailable, a deterministic
    fallback still produces an editable suggestion.

  Details: [Script Import (Fountain)](/cms/script-import).
</Update>

<Update label="2026-07-02" description="Format 1.1" tags={["feature"]}>
  ## Format 1.1 — video panels

  Additive, backward-compatible release: every valid 1.0 manifest remains valid unchanged.

  - **Video playback control** on video layers: `playMode` (`once`, `loop`, `pingpong`, `loop-from`), `startMode` (`on-view`, `on-hover`, `on-click`), `loopFromMs`, and native `controls`.
  - **Work-level video defaults** in `settings.ui` (`videoPlayModeDefault`, `videoStartModeDefault`, `videoMutedDefault`), cascading to per-layer overrides.
  - **Video variants** can mark a playback `direction` (`forward`/`reverse`) for pre-rendered reverse encodes used by smooth ping-pong playback.
  - **Poster frames** on video catalog items for click-to-play and reduced-motion presentation.
  - **Tracking** event whitelist extended with video playback events.
  - The 1.0 fields `autoplay`/`loop` on video layers remain valid; the new fields take precedence when both are present.

  Details: [Video schema reference](/schema/video) and [Versioning](/schema/versioning).
</Update>

<Update label="Format 1.0" description="Initial specification" tags={["feature"]}>
  ## Format 1.0 — initial public specification

  First public release of the PanelWave manifest format (JSON Schema 2020-12, CC BY 4.0):

  - Chapters, pages, and panels with layered content (image, video, text, audio, plugin layers)
  - Graph-based flow: entry panels, edges, transitions, and JSON Logic conditions
  - Asset catalog with image/audio/video/subtitle variants
  - Typed variable system with five scopes
  - Localization via `LocalizedString`, BCP-47 locales, and per-locale asset variants
  - Output format presets for web, print, and video
  - Paywall and entitlement rules, tracking configuration, plugin support, accessibility hints
  - `x-` extension fields throughout

  Details: [Schema overview](/schema/overview).
</Update>
